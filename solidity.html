Solidity syntax

Solidity allows you to write smart contracts in the blockchain, the syntax is very similar to javascript, but just the syntaxis, because solidity ends up being a limited programming compile language dedicated to specific usages.

The structure starts with SPDX license identifier, then the pragma statement that declares the version that you are going to use.

Type of variables

Fixed-size in memory, Variable size memory, this one is used when you are not sure about the size of the data, and user-defined data for costume the information.

Examples for the fixed size memory:

  • boolean, can be very useful and it can be equal to true or false.
  • uint that stands for unsigned integer, this is very useful for transfer finances.
  • address is useful for financial tranfers to name the recipient.
  • Finally the bytes32 very used to manipulate strings in validations, for example to avoid content hevier than 32 bytes.

Variable size memory

  • string is can be useful to name data.

Main variables

These are the must useful variables.

¿What is a constructor?

Is a function that can only be one time after the deploy of the smart contract, where you iniatialize a variable.

Functions

To manipulate your data is important to know how to create functions, you can create a value and then get it with a "getValue()", also use the return keyword to return the value, and inside of it to use add another return the final value. The set function modifies data in the blockchain and the get only reads it.

Note

The set function makes you pay a transaction fee of gas and the get function doesn't.

Function visibility keywords

Function visibility keywords allow you to manage the access to the functions. We must remmember that saying that a variable can't be read outside the smart contract is only in the context of the Ethereum Virtual Machine, because in the blockchain everything is public.

  • private, to call the function only from the inside of the smart contract.
  • internal, this one can't be called from outside the contract either, but it can be called from a smart contract who inherit from the smart contract.
  • external, this one can't be called from inside the smart contract.
  • public, with this one you can call the function from inside and outside the smart contract.

Structure Controls

Solidity shares many of the same controls structures used in others classic programming languages like javascript or python.

  • if, for conditions.
  • for, for create a loop.
  • while, the difference between the while and the for is that you are not sure how many times is going to run.

How to create and manipulate arrays

There is two type of arrays. First you need to declare the array and remmember you cannot manipulate arrays outside of a function.

  • Storage Arrays, these arrays are stored in the blockchains, there is dinamic size and fixed size in storage arrays.
  • Memory Arrays, these arrays only store while they are being executed, and the size has to be specified

In this arrays there is 5 type of oparations that you can apply:

  • crud
  • create
  • read
  • update
  • delete

Mapping

With mapping is much eassier to retrieve a specific value. Mapping will allow you to make approvals, like when information through keys.

Struct

Are made to structurate the data with many fields. First we need to declare struct, then we should create instances for the storage of the structure, then we can create an array of it, or do a mapping of it.

Events

With events you can bring data from the smart contract to the frontend, first you need to declare the event and then you need to emit it inside of a function, the indexed keyword can be used for listen to a specific, but you must remmember that using a lot of indexed can make your smart contract quite expensive.

How to call functions of other smart contracts in solidity?

The first thing is to get the interface of the other contract, then we need the adress of the other contract. You must remmember that if the function that you are calling has mistakes that will make an error propagation.

How to send Ether from a smart contract?

Is very important to understan how the manipulation works, to send ether we need another adress, it could be antoher smart contract.The function specifies from wich address to send how much ether. To recieve ether you need to put the keyword payable.

Keywords for handling errors in smart contracts

  • revert this one is for errors that usually happens.
  • require() this is for errors that should never happen.
  • assert() which asserts that a condition is met during the computation process inside a function.

Function modifiers

Allows you to run sume code before the execution of a function, this is good for validating arguments, like an access control. First we need to create a function, then we should create a modifier with the keyword modifier. You can only call the modifier from inside the smart contract. Then we need to attach the modifer to the function, so the modifer can be executed first and validate his argument. And even we could add more than one modifier.

Inheritance

This allows you to reuse some code. This could work by having to files, and asigning a parent and a child who is goin to inherit some code, this one could also manipulate the data calling the constructor. And the parent's constructor will be executed before the child's constructor. It is also important to say that you can have multiple inheritance.
